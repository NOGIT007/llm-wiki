Server
Bun HTTP Server
Single-file TypeScript server (server.ts, ~740 lines). Serves the UI, exposes REST API, parses YAML frontmatter, extracts [[wikilinks]], LLM chat integration, multi-vault support, and GCS backup. Auto-reloads wiki data every 5 seconds.

Bash control script for server lifecycle. Manages PID file at /tmp/llm-wiki-server.pid. Commands: start, stop, restart, status.

Data (Filesystem)
wiki/
LLM-authored markdown pages with YAML frontmatter (title, type, tags, sources). Types: source, entity, concept, analysis, comparison. Cross-linked via [[wikilinks]].

Immutable source material added by the human. Never modified. Ingested by the LLM into wiki pages. Can be added via UI (paste, upload, URL fetch).

Config Files
.wiki-queue.json holds pending actions (ingest, fix-issues) queued from the UI. .wiki-config.json stores settings like autoIngest toggle.
queue config
Claude CLI reads raw/ → writes wiki/
Intelligence Layer
Claude (LLM)
The actual "brain" of the wiki. Runs via Claude Code CLI. Reads instructions from CLAUDE.md, processes the ingest/query/lint queue, writes all wiki content, manages cross-references, and maintains the index and changelog.
Claude Code CLI CLAUDE.md markdown I/O

Core Workflows

1Ingest

  • Human adds source to raw/ (paste, upload, or URL)
  • UI queues ingest request to .wiki-queue.json
  • Claude reads queue on conversation start
  • Reads raw source, discusses takeaways
  • Creates src-*.md summary page in wiki/
  • Updates or creates entity/concept pages
  • Adds [[wikilinks]] across all affected pages
  • Updates index.md and log.md

2Query

  • Human asks a question about wiki knowledge
  • Claude reads index.md to find relevant pages
  • Reads pages, synthesizes an answer
  • Cites sources with [[wikilinks]]
  • If substantial: offers to file as analysis page
  • Updates index and log if new page created

3Lint

  • Human requests health check
  • Claude scans all wiki pages
  • Finds contradictions, stale claims, orphans
  • Checks for missing cross-references
  • Identifies concepts needing their own page
  • Fixes what it can, reports the rest
  • Suggests new sources to investigate

Design Decisions

No Database
The filesystem IS the database. Markdown files are the source of truth. Git provides versioning. The server just reads and indexes them in-memory.
No Build Step
Zero bundlers, zero frameworks. One HTML file with inline CSS/JS. Bun runs TypeScript natively. bun run server.ts and you're live.
LLM-Native
The wiki is designed from the ground up to be written by an LLM. CLAUDE.md is the operating manual. The queue bridges UI actions to LLM processing. Markdown + YAML frontmatter is the lingua franca.
Obsidian-Compatible
[[Wikilinks]], YAML frontmatter, and flat markdown files mean the wiki/ folder can be opened directly in Obsidian for an alternative view.
